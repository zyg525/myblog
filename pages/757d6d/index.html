使用Dockerfile为SpringBoot应用构建Docker镜像

学习不走弯路，关注公众号 回复「学习路线」，获取mall项目专属学习路线！

# 使用Dockerfile为SpringBoot应用构建Docker镜像

上次写过一篇使用Maven插件构建Docker镜像 (opens new window) ，讲述了通过docker-maven-plugin来构建docker镜像的方式，此种方式需要依赖自建的Registry镜像仓库。本文将讲述另一种方式，使用Dockerfile来构建docker镜像，此种方式不需要依赖自建的镜像仓库，只需要应用的jar包和一个Dockerfile文件即可。

# Dockerfile常用指令

# ADD

用于复制文件，格式：

ADD <src> <dest>
1

示例：

# 将当前目录下的mall-tiny-docker-file.jar包复制到docker容器的/目录下
ADD mall-tiny-docker-file.jar /mall-tiny-docker-file.jar
1
2

# ENTRYPOINT

指定docker容器启动时执行的命令，格式：

ENTRYPOINT ["executable", "param1","param2"...]
1

示例：

# 指定docker容器启动时运行jar包
ENTRYPOINT ["java", "-jar","/mall-tiny-docker-file.jar"]
1
2

# ENV

用于设置环境变量，格式：

ENV <key> <value>
1

示例：

# mysql运行时设置root密码
ENV MYSQL_ROOT_PASSWORD root
1
2

# EXPOSE

声明需要暴露的端口(只声明不会打开端口)，格式：

EXPOSE <port1> <port2> ...
1

示例：

# 声明服务运行在8080端口
EXPOSE 8080
1
2

# FROM

指定所需依赖的基础镜像，格式：

FROM <image>:<tag>
1

示例：

# 该镜像需要依赖的java8的镜像
FROM java:8
1
2

# MAINTAINER

指定维护者的名字，格式：

MAINTAINER <name>
1

示例：

MAINTAINER macrozheng
1

# RUN

在容器构建过程中执行的命令，我们可以用该命令自定义容器的行为，比如安装一些软件，创建一些文件等，格式：

RUN <command>
RUN ["executable", "param1","param2"...]
1
2

示例：

# 在容器构建过程中需要在/目录下创建一个mall-tiny-docker-file.jar文件
RUN bash -c 'touch /mall-tiny-docker-file.jar'
1
2

# 使用Dockerfile构建SpringBoot应用镜像

# 编写Dockerfile文件

# 该镜像需要依赖的基础镜像
FROM java:8
# 将当前目录下的jar包复制到docker容器的/目录下
ADD mall-tiny-docker-file-0.0.1-SNAPSHOT.jar /mall-tiny-docker-file.jar
# 运行过程中创建一个mall-tiny-docker-file.jar文件
RUN bash -c 'touch /mall-tiny-docker-file.jar'
# 声明服务运行在8080端口
EXPOSE 8080
# 指定docker容器启动时运行jar包
ENTRYPOINT ["java", "-jar","/mall-tiny-docker-file.jar"]
# 指定维护者的名字
MAINTAINER macrozheng

# 运行mysql服务并设置

# 1.使用docker命令启动：

docker run -p 3306:3306 --name mysql \
-v /mydata/mysql/log:/var/log/mysql \
-v /mydata/mysql/data:/var/lib/mysql \
-v /mydata/mysql/conf:/etc/mysql \
-e MYSQL_ROOT_PASSWORD=root  \
-d mysql:5.7
1
2
3
4
5
6

# 2.进入运行mysql的docker容器：

docker exec -it mysql /bin/bash
1

# 3.使用mysql命令打开客户端：

mysql -uroot -proot --default-character-set=utf8
1

# 4.修改root帐号的权限，使得任何ip都能访问：

grant all privileges on *.* to 'root'@'%'
1

# 5.创建mall数据库：

create database mall character set utf8
1

# 6.将mall.sql文件拷贝到mysql容器的/目录下：

docker cp /mydata/mall.sql mysql:/
1

# 7.将sql文件导入到数据库：

use mall;
source /mall.sql;
1
2

# 运行mall-tiny-docker-file应用

docker run -p 8080:8080 --name mall-tiny-docker-file \
--link mysql:db \
-v /etc/localtime:/etc/localtime \
-v /mydata/app/mall-tiny-docker-file/logs:/var/logs \
-d mall-tiny/mall-tiny-docker-file:0.0.1-SNAPSHOT
